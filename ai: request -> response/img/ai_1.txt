Полная схема: от запроса пользователя до ответа модели

╔══════════════════════════════════════════════════════════════════╗
║  1. ВВОД ПОЛЬЗОВАТЕЛЯ                                          ║
║     "Расскажи анекдот про кота"                                 ║
╚════════════════════════════╤═════════════════════════════════════╝
                             │
                             ▼
╔══════════════════════════════════════════════════════════════════╗
║  2. ТОКЕНИЗАЦИЯ                                                 ║
║                                                                  ║
║  Текст разбивается на токены по словарю модели (BPE алгоритм)   ║
║                                                                  ║
║  "Расскажи анекдот про кота"                                    ║
║       ↓                                                          ║
║  ["Рас", "скажи", " анек", "дот", " про", " кота"]             ║
║                                                                  ║
║  Каждый токен получает числовой ID из словаря:                   ║
║  [4821, 9903, 7712, 3304, 1082, 8845]                           ║
╚════════════════════════════╤═════════════════════════════════════╝
                             │
                             ▼
╔══════════════════════════════════════════════════════════════════╗
║  3. ЭМБЕДДИНГ (Embedding)                                       ║
║                                                                  ║
║  Каждый token ID превращается в вектор — массив чисел,          ║
║  который кодирует СМЫСЛ токена в многомерном пространстве       ║
║                                                                  ║
║  ID 4821 → [0.12, -0.45, 0.78, 0.03, ..., -0.21]  (2048+ чисел)║
║  ID 9903 → [0.56, 0.11, -0.33, 0.89, ..., 0.44]                ║
║  ...                                                             ║
║                                                                  ║
║  + Positional Encoding — добавляется информация о ПОЗИЦИИ       ║
║    каждого токена в предложении (модель должна знать порядок)    ║
╚════════════════════════════╤═════════════════════════════════════╝
                             │
                             ▼
╔══════════════════════════════════════════════════════════════════╗
║  4. ТРАНСФОРМЕР (Transformer)                                    ║
║                                                                  ║
║  Ядро модели. Набор слоёв (у Gemini — десятки слоёв),           ║
║  каждый слой содержит:                                           ║
║                                                                  ║
║  ┌─────────────────────────────────────────────────────────┐    ║
║  │  Self-Attention (механизм внимания)                      │    ║
║  │                                                          │    ║
║  │  Каждый токен "смотрит" на все остальные токены           │    ║
║  │  и определяет, какие из них важны для текущего контекста  │    ║
║  │                                                          │    ║
║  │  "кота" обращает внимание на "анекдот" и "расскажи"      │    ║
║  │  → понимает: нужен юмор про кота                         │    ║
║  └─────────────────────────────────────────────────────────┘    ║
║                         │                                        ║
║                         ▼                                        ║
║  ┌─────────────────────────────────────────────────────────┐    ║
║  │  Feed-Forward Network (нейросеть прямого прохода)        │    ║
║  │                                                          │    ║
║  │  Обрабатывает результат attention, извлекает              │    ║
║  │  более сложные закономерности и "знания"                 │    ║
║  └─────────────────────────────────────────────────────────┘    ║
║                                                                  ║
║  Это повторяется N раз (слой за слоем), каждый раз углубляя     ║
║  понимание контекста                                             ║
╚════════════════════════════╤═════════════════════════════════════╝
                             │
                             ▼
╔══════════════════════════════════════════════════════════════════╗
║  5. ЛОГИТЫ (Logits)                                              ║
║                                                                  ║
║  На выходе трансформера — один вектор, который проецируется      ║
║  на ВЕСЬ словарь. Получаем сырую оценку для каждого токена:     ║
║                                                                  ║
║  "Идёт"    →  5.8                                                ║
║  "Жил"     →  5.2                                                ║
║  "Кот"     →  4.1                                                ║
║  "Однажды" →  3.9                                                ║
║  "The"     → -2.1                                                ║
║  "xyz"     → -9.4                                                ║
║  ... (30 000+ оценок — по одной на каждый токен в словаре)      ║
║                                                                  ║
║  Это ещё НЕ вероятности, просто числа                            ║
╚════════════════════════════╤═════════════════════════════════════╝
                             │
                             ▼
╔══════════════════════════════════════════════════════════════════╗
║  6. СЭМПЛИРОВАНИЕ (выбор следующего токена)                      ║
║                                                                  ║
║  6.1  Temperature                                                ║
║  ─────────────────                                               ║
║  Каждый логит делится на temperature:                             ║
║  logit / temperature                                             ║
║                                                                  ║
║  Низкая (0.1) → разница усиливается → предсказуемый ответ       ║
║  Высокая (2.0) → разница сглаживается → креативный ответ        ║
║                         │                                        ║
║                         ▼                                        ║
║  6.2  Top-K                                                      ║
║  ─────────────────                                               ║
║  Оставляем только K самых вероятных токенов                      ║
║  (topK=64 → берём 64, остальные 29 936 отбрасываем)             ║
║                         │                                        ║
║                         ▼                                        ║
║  6.3  Top-P (Nucleus Sampling)                                   ║
║  ─────────────────                                               ║
║  Из оставшихся берём минимальный набор, чья суммарная            ║
║  вероятность ≥ 95% (topP=0.95)                                   ║
║                         │                                        ║
║                         ▼                                        ║
║  6.4  Softmax → Вероятности                                      ║
║  ─────────────────                                               ║
║  Преобразуем отфильтрованные логиты в вероятности:               ║
║                                                                  ║
║  "Идёт"    → 42%                                                 ║
║  "Жил"     → 31%                                                 ║
║  "Кот"     → 18%                                                 ║
║  "Однажды" →  9%                                                 ║
║                         │                                        ║
║                         ▼                                        ║
║  6.5  Случайный выбор (sampling)                                 ║
║  ─────────────────                                               ║
║  Бросаем "кубик" с учётом вероятностей                           ║
║  Результат: "Идёт"                                               ║
╚════════════════════════════╤═════════════════════════════════════╝
                             │
                             ▼
╔══════════════════════════════════════════════════════════════════╗
║  7. ПРОВЕРКА ОСТАНОВКИ                                           ║
║                                                                  ║
║  Сгенерированный токен = EOS (конец)?     → СТОП                ║
║  Достигнут maxOutputTokens?                → СТОП                ║
║  Встретился stop sequence?                 → СТОП                ║
║  Ничего из этого?                          → ПРОДОЛЖАЕМ ↓       ║
╚════════════════════════════╤═════════════════════════════════════╝
                             │
                             ▼
╔══════════════════════════════════════════════════════════════════╗
║  8. АВТОРЕГРЕССИОННЫЙ ЦИКЛ                                       ║
║                                                                  ║
║  Выбранный токен добавляется к входной последовательности:       ║
║                                                                  ║
║  Итерация 1: "Расскажи анекдот про кота"          → "Идёт"     ║
║  Итерация 2: "Расскажи анекдот про кота Идёт"     → "кот"      ║
║  Итерация 3: "Расскажи анекдот про кота Идёт кот" → "по"       ║
║  ...                                                             ║
║                                                                  ║
║  Каждая итерация проходит ЗАНОВО шаги 3 → 4 → 5 → 6 → 7       ║
║  (с учётом KV-cache для ускорения)                               ║
╚════════════════════════════╤═════════════════════════════════════╝
                             │
                             ▼
╔══════════════════════════════════════════════════════════════════╗
║  9. ДЕТОКЕНИЗАЦИЯ                                                ║
║                                                                  ║
║  Набранные токены склеиваются обратно в текст:                   ║
║                                                                  ║
║  ["Идёт", " кот", " по", " крыше", "..."]                      ║
║       ↓                                                          ║
║  "Идёт кот по крыше..."                                          ║
║                                                                  ║
║  Этот текст отдаётся пользователю                                ║
║  (при streaming — токен за токеном по мере генерации)            ║
╚══════════════════════════════════════════════════════════════════╝
