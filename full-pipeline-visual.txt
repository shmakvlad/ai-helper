  Пример запроса: "Столица Франции — "
  Ожидаемый ответ: "Париж"

  Модель: 32 слоя, d_model = 4096, 32 головы, словарь = 100 000 токенов

┃  ЭТАП 2. ТОКЕНИЗАЦИЯ                                                         ┃
┃  Текст → числа                                                               ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

  ╔═════════════════════════════════════════════════════════════════════════╗
  ║  АНАЛОГИЯ: Книга с номерами вместо слов                               ║
  ║                                                                       ║
  ║  Представь словарь, где каждому фрагменту текста присвоен номер:     ║
  ║  "кот" = 4821, "собака" = 7390, "Столица" = 2841, " Фран" = 9012   ║
  ║  Токенизация — это замена текста на номера по этому словарю.          ║
  ╚═════════════════════════════════════════════════════════════════════════╝

  Алгоритм BPE (Byte-Pair Encoding) разбивает текст на фрагменты
  (токены) и находит их номера (ID) в словаре модели:

  "Столица Франции — "
       │
       ▼
  ┌──────────┬──────────┬──────────┬──────────┬──────────┐
  │ "Столица"│ " Фран"  │ "ции"    │ " —"     │ " "      │
  │          │          │          │          │          │
  │ ID: 2841 │ ID: 9012 │ ID: 4455 │ ID: 1087 │ ID: 220  │
  └──────────┴──────────┴──────────┴──────────┴──────────┘
      t₁          t₂         t₃         t₄         t₅

  Результат: вектор из 5 целых чисел

  tokens = [2841, 9012, 4455, 1087, 220]

  Почему не целые слова? Потому что:
  • Словарь конечен (~100 000 записей) — всех слов не покрыть.
  • Редкие слова разбиваются на частые подслова:
    "трансформер" → "транс" + "форм" + "ер"
  • Это позволяет обрабатывать ЛЮБОЙ текст, даже несуществующие слова.

┃  ЭТАП 3. ЭМБЕДДИНГИ                                                          ┃
┃  Числа → векторы смысла                                                       ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

  ╔═════════════════════════════════════════════════════════════════════════╗
  ║  АНАЛОГИЯ: GPS-координаты для слов                                    ║
  ║                                                                       ║
  ║  ID — это как почтовый индекс: 101000. Сам по себе ничего не         ║
  ║  говорит о месте. Но GPS-координаты (55.75, 37.62) — это точка      ║
  ║  на карте. Москва и Петербург — рядом. Москва и Токио — далеко.     ║
  ║                                                                       ║
  ║  Эмбеддинг — это «GPS-координаты» слова в пространстве смыслов.     ║
  ║  Только измерений не 2 (широта, долгота), а 4096.                    ║
  ╚═════════════════════════════════════════════════════════════════════════╝

  Операция: поиск в таблице (lookup)

  Таблица эмбеддингов E — матрица размером vocab_size × d_model:
  E ∈ ℝ^(100000 × 4096)

  Каждая строка — вектор одного токена:

  ┌─────────────────────────────────────────────────────────────────────┐
  │                                                                     │
  │  Таблица E (100 000 строк × 4096 столбцов)                        │
  │                                                                     │
  │  Строка 0:    [0.021, −0.445, 0.112, ..., 0.033]  ← токен 0      │
  │  Строка 1:    [0.187, 0.023, −0.891, ..., −0.456] ← токен 1      │
  │  ...                                                                │
  │  Строка 2841: [0.512, −0.178, 0.634, ..., 0.291]  ← "Столица"    │
  │  ...                                                                │
  │  Строка 9012: [0.087, 0.923, −0.345, ..., 0.178]  ← " Фран"      │
  │  ...                                                                │
  │  Строка 99999:[−0.334, 0.112, 0.067, ..., −0.890] ← токен 99999  │
  │                                                                     │
  └─────────────────────────────────────────────────────────────────────┘

  Для каждого ID берём соответствующую строку:

  ID 2841 → E[2841] = [0.512, −0.178, 0.634, ..., 0.291]   (4096 чисел)
  ID 9012 → E[9012] = [0.087,  0.923, −0.345, ..., 0.178]  (4096 чисел)
  ID 4455 → E[4455] = [−0.234, 0.456,  0.123, ..., −0.567] (4096 чисел)
  ID 1087 → E[1087] = [0.345, −0.678,  0.890, ..., 0.012]  (4096 чисел)
  ID 220  → E[220]  = [−0.111, 0.222, −0.333, ..., 0.444]  (4096 чисел)

  Результат: матрица X₀ размером 5 × 4096

  ┌───────┬─────────────────────────────────────────────────────────┐
  │ Токен │                 Вектор (4096 чисел)                     │
  ├───────┼─────────────────────────────────────────────────────────┤
  │  t₁   │ [ 0.512, −0.178,  0.634, ...,  0.291]  ← "Столица"   │
  │  t₂   │ [ 0.087,  0.923, −0.345, ...,  0.178]  ← " Фран"     │
  │  t₃   │ [−0.234,  0.456,  0.123, ..., −0.567]  ← "ции"        │
  │  t₄   │ [ 0.345, −0.678,  0.890, ...,  0.012]  ← " —"         │
  │  t₅   │ [−0.111,  0.222, −0.333, ...,  0.444]  ← " "          │
  └───────┴─────────────────────────────────────────────────────────┘
     5 строк                                          4096 столбцов

  ПРОБЛЕМА: "Столица Франции" и "Франции Столица" дали бы одинаковые
  векторы — модель не знает порядок слов!

  ╔═════════════════════════════════════════════════════════════════════════╗
  ║  АНАЛОГИЯ: Номера мест в кинотеатре                                   ║
  ║                                                                       ║
  ║  Ты знаешь, КТО пришёл (эмбеддинг), но не знаешь, ГДЕ сидит.       ║
  ║  Позиционное кодирование — это билет с номером ряда и места.         ║
  ║  Теперь ясно: Маша на месте 1, Петя на месте 2.                     ║
  ╚═════════════════════════════════════════════════════════════════════════╝

  К каждому вектору ПРИБАВЛЯЕТСЯ вектор позиции:

  Формула: X = Embedding(token) + PositionalEncoding(position)

  Вектор позиции pos₁:   [0.010, −0.005,  0.020, ..., 0.015]
  Вектор позиции pos₂:   [0.008,  0.012, −0.003, ..., 0.009]
  ...и так для каждой позиции

  Для позиции pos и измерения i (оригинальная формула из статьи):
  PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
  PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

  (Современные модели часто используют обучаемые позиции или RoPE)

  Сложение:

  Эмбеддинг "Столица":   [ 0.512, −0.178,  0.634, ...]
  + Позиция 1:           [ 0.010, −0.005,  0.020, ...]
  ─────────────────────────────────────────────────────
  = Итого:               [ 0.522, −0.183,  0.654, ...]
                           ↑
                     слово + его позиция

  Результат: матрица X ∈ ℝ^(5 × 4096)
  Каждая строка теперь кодирует И смысл, И позицию токена.

  ┌───────┬───────────────────────────────────────────────────────────────┐
  │ Токен │            Эмбеддинг + Позиция                               │
  ├───────┼───────────────────────────────────────────────────────────────┤
  │  t₁   │ [ 0.522, −0.183,  0.654, ...,  0.306]  ← "Столица" + pos₁ │
  │  t₂   │ [ 0.095,  0.935, −0.348, ...,  0.187]  ← " Фран" + pos₂   │
  │  t₃   │ [−0.228,  0.462,  0.132, ..., −0.558]  ← "ции" + pos₃     │
  │  t₄   │ [ 0.354, −0.670,  0.899, ...,  0.023]  ← " —" + pos₄      │
  │  t₅   │ [−0.100,  0.233, −0.320, ...,  0.457]  ← " " + pos₅       │
  └───────┴───────────────────────────────────────────────────────────────┘

  Это ВХОД в трансформер. Дальше — самое интересное.

  ║                                                                       ║
  ║  Представь завод с 32 станциями на конвейере. На вход поступает       ║
  ║  заготовка (наши векторы). Каждая станция:                            ║

  ┌─────────────────────────────────────────────────────────────────────┐
  │                                                                     │
  │  Вход: X ∈ ℝ^(5 × 4096)                                           │
  │    │                                                                │

  │  ┌──────────────────────────┐       │                               │
  │  │  Multi-Head Attention    │       │                               │
  │  │  (32 головы)             │       │                               │
  │  └──────┬───────────────────┘       │                               │
  │         ▼                            │                               │
  │  ┌──────────────┐                   │                               │
  │  │     ⊕ ADD    │ ◄────────────────┘                               │
  │  └──────┬───────┘                                                   │
  │         │                                                           │
  │         ├────────────────────────────┐                               │
  │         ▼                            │ (skip connection)             │
  │  ┌──────────────┐                   │                               │
  │  │  Layer Norm   │                   │                               │
  │  └──────┬───────┘                   │                               │

  │  │  4096 → 16384 → 4096    │       │                               │
  │  └──────┬───────────────────┘       │                               │

  │  ┌──────────────┐                   │                               │

  X₀ → [Слой 1] → X₁ → [Слой 2] → X₂ → ... → [Слой 32] → X₃₂

  Меняется СОДЕРЖАНИЕ — векторы становятся всё более «умными».

  ШАГ 5A.1: Создание Q, K, V
  ───────────────────────────

  Три матрицы весов (параметры, выученные при обучении):
  W_Q ∈ ℝ^(4096 × 4096)
  W_K ∈ ℝ^(4096 × 4096)
  W_V ∈ ℝ^(4096 × 4096)

  Формулы:
  Q = X · W_Q     (что каждый токен ИЩЕТ)
  K = X · W_K     (что каждый токен ПРЕДЛАГАЕТ)
  V = X · W_V     (что каждый токен ОТДАЁТ)

  Все три — матрицы 5 × 4096:

  ┌──────────────────────────────────────────────────────────────────┐
  │                                                                  │
  │  X (5×4096)  ×  W_Q (4096×4096)  =  Q (5×4096)                 │
  │                                                                  │
  │   "Столица"  ──→  Q₁ = "Ищу: что является столицей?"           │
  │   " Фран"    ──→  Q₂ = "Ищу: какая страна?"                    │
  │   "ции"      ──→  Q₃ = "Ищу: окончание чего?"                  │
  │   " —"       ──→  Q₄ = "Ищу: что следует после тире?"          │
  │   " "        ──→  Q₅ = "Ищу: что будет дальше?"                │
  │                                                                  │
  │  (Аналогично для K и V)                                         │
  │                                                                  │
  └──────────────────────────────────────────────────────────────────┘


  ШАГ 5A.2: Вычисление оценок внимания
  ─────────────────────────────────────

  Каждый токен сравнивает свой Q с K всех остальных:

  Формула: Score = Q · K^T

  Score ∈ ℝ^(5 × 5) — матрица «кто на кого смотрит»:

  Q₁ · K₁ = ?    Q₁ · K₂ = ?    Q₁ · K₃ = ?    ...

  Скалярное произведение двух векторов:
  Q₁ · K₂ = q₁k₁ + q₂k₂ + q₃k₃ + ... + q₄₀₉₆k₄₀₉₆

  Чем больше результат → тем больше сходство → тем сильнее внимание.


  ШАГ 5A.3: Масштабирование
  ──────────────────────────

  Score_scaled = Score / √d_k

  d_k = 4096 (размерность ключа)
  √4096 = 64

  Score_scaled = Score / 64

  ╔═════════════════════════════════════════════════════════════════════════╗
  ║  ЗАЧЕМ ДЕЛИТЬ?                                                        ║
  ║                                                                       ║
  ║  При скалярном произведении двух длинных векторов (4096 чисел)        ║
  ║  результат может быть огромным (например 500). Softmax от 500         ║
  ║  даст практически 1.0 для одного элемента и 0.0 для всех остальных. ║
  ║  Модель будет «смотреть» только на один токен, игнорируя контекст.   ║
  ║  Деление на √d_k сжимает числа в разумный диапазон.                  ║
  ╚═════════════════════════════════════════════════════════════════════════╝


  ШАГ 5A.4: Маска (для декодера)
  ──────────────────────────────

  В декодерах (GPT, Claude) токены НЕ МОГУТ смотреть в будущее.
  Накладывается маска — будущие позиции заменяются на −∞:

  ┌───────────────────────────────────────────────────────────────┐
  │                                                               │
  │  До маски:                  После маски:                      │
  │                                                               │
  │      t₁   t₂   t₃   t₄   t₅       t₁    t₂    t₃   t₄  t₅ │
  │  t₁ [2.1  0.8  0.3  0.1  0.5]  t₁ [2.1   −∞    −∞   −∞  −∞]│
  │  t₂ [1.2  3.4  0.7  0.2  0.1]  t₂ [1.2   3.4   −∞   −∞  −∞]│
  │  t₃ [0.5  2.1  1.8  0.4  0.3]  t₃ [0.5   2.1   1.8  −∞  −∞]│
  │  t₄ [0.3  1.5  0.9  2.7  0.6]  t₄ [0.3   1.5   0.9  2.7 −∞]│
  │  t₅ [0.8  2.3  1.1  1.9  0.4]  t₅ [0.8   2.3   1.1  1.9 0.4│
  │                                                               │
  │  −∞ после softmax → 0 (токен не получает внимания)           │
  │                                                               │
  └───────────────────────────────────────────────────────────────┘


  ШАГ 5A.5: Softmax → веса внимания
  ──────────────────────────────────

  Формула softmax:
  softmax(xᵢ) = eˣⁱ / Σⱼ eˣʲ

  Превращает сырые числа в вероятности (0..1), сумма по строке = 1:

  ┌──────────────────────────────────────────────────────────────────┐
  │                                                                  │
  │  Матрица весов внимания A (после softmax):                      │
  │                                                                  │
  │          "Столица"  " Фран"   "ции"    " —"     " "             │
  │                                                                  │
  │  "Столица"  1.00     0.00     0.00     0.00    0.00             │
  │  " Фран"    0.30     0.70     0.00     0.00    0.00             │
  │  "ции"      0.15     0.55     0.30     0.00    0.00             │
  │  " —"       0.10     0.25     0.15     0.50    0.00             │
  │  " "        0.12     0.35     0.18     0.25    0.10             │
  │                                                                  │
  │  Сумма каждой строки = 1.00                                     │
  │                                                                  │
  └──────────────────────────────────────────────────────────────────┘

  Читаем: токен " " (позиция 5 — последний, тот что предскажет
  следующее слово) сильнее всего смотрит на " Фран" (0.35) и
  " —" (0.25). Модель «понимает»: речь о Франции, нужен ответ
  после тире.


  ШАГ 5A.6: Взвешенная сумма Value
  ─────────────────────────────────

  Формула: Output = A · V

  Каждый токен получает НОВЫЙ вектор — смесь Value всех
  токенов, на которые он смотрел, с весами из матрицы A:

  output₅ = 0.12·V₁ + 0.35·V₂ + 0.18·V₃ + 0.25·V₄ + 0.10·V₅

  Токен " " впитал информацию от всех предыдущих, особенно
  от " Фран" — теперь его вектор «знает» про Францию.


  ПОЛНАЯ ФОРМУЛА ATTENTION:
  ┌──────────────────────────────────────────────────────────────────┐
  │                                                                  │
  │  Attention(Q, K, V) = softmax( Q · K^T / √d_k + Mask ) · V    │
  │                                                                  │
  │  Размерности:                                                    │
  │  Q: (5 × 4096)                                                   │
  │  K^T: (4096 × 5)                                                 │
  │  Q · K^T: (5 × 5)  — матрица внимания                          │
  │  / √4096 = / 64                                                  │
  │  softmax: (5 × 5) → (5 × 5) вероятности                        │
  │  × V: (5 × 5) · (5 × 4096) = (5 × 4096)                       │
  │                                                                  │
  │  Вход: (5 × 4096) → Выход: (5 × 4096)                          │
  │  Размерность не меняется!                                        │
  │                                                                  │
  └──────────────────────────────────────────────────────────────────┘

┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
┃  ЭТАП 5B. MULTI-HEAD ATTENTION — 32 ПАРАЛЛЕЛЬНЫХ ВЗГЛЯДА                     ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

  ╔═════════════════════════════════════════════════════════════════════════╗
  ║  АНАЛОГИЯ: 32 эксперта смотрят один документ                          ║
  ║                                                                       ║
  ║  Один эксперт проверяет грамматику, другой — смысл,                  ║
  ║  третий — стиль, четвёртый — факты. Потом собираются                 ║
  ║  и составляют общее заключение.                                       ║
  ╚═════════════════════════════════════════════════════════════════════════╝

  Вместо одного attention с d=4096, запускаем 32 головы с d=128:
  d_head = d_model / n_heads = 4096 / 32 = 128

  Каждая голова имеет свои W_Q, W_K, W_V размером 4096 × 128:

  ┌──────────────────────────────────────────────────────────────────┐
  │                                                                  │
  │  Вход X (5 × 4096)                                              │
  │    │                                                             │
  │    ├──→ Голова 1: Q₁K₁V₁ (5×128) → Attention → out₁ (5×128)   │
  │    ├──→ Голова 2: Q₂K₂V₂ (5×128) → Attention → out₂ (5×128)   │
  │    ├──→ Голова 3: Q₃K₃V₃ (5×128) → Attention → out₃ (5×128)   │
  │    ├──→  ...                                                     │
  │    └──→ Голова 32: Q₃₂K₃₂V₃₂     → Attention → out₃₂ (5×128) │
  │                                                                  │
  │    Конкатенация: [out₁ ; out₂ ; ... ; out₃₂]                   │
  │    = (5 × 128×32) = (5 × 4096)                                  │
  │                                                                  │
  │    Проекция: × W_O (4096 × 4096)                                │
  │    = (5 × 4096)                                                  │
  │                                                                  │
  └──────────────────────────────────────────────────────────────────┘

  Формула:
  MultiHead(Q,K,V) = Concat(head₁, ..., head₃₂) · W_O
  где headᵢ = Attention(X·W_Qⁱ, X·W_Kⁱ, X·W_Vⁱ)

  Что видит каждая голова (примеры):
  • Голова 3:  "ции" → " Фран" (грамматическая связь: окончание)
  • Голова 12: " —" → "Столица" (семантика: тире после определения)
  • Голова 25: " " → " Фран" + "ции" (тема: Франция — ключевое)

┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
┃  ЭТАП 5C. RESIDUAL CONNECTION + LAYER NORM                                    ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

  ╔═════════════════════════════════════════════════════════════════════════╗
  ║  АНАЛОГИЯ: Записная книжка + редактор                                 ║
  ║                                                                       ║
  ║  Skip connection — ты записываешь новую информацию НА ПОЛЯХ,         ║
  ║  а не ВМЕСТО старого текста. Если новая заметка бесполезна —         ║
  ║  оригинал сохранён.                                                   ║
  ║  Layer Norm — редактор, который выравнивает почерк (числа),          ║
  ║  чтобы следующий читатель мог разобрать.                              ║
  ╚═════════════════════════════════════════════════════════════════════════╝

  Формулы:

  X_attn = LayerNorm(X + MultiHeadAttention(X))
                      ↑       ↑
                  оригинал  новое (skip connection: сложение)

  LayerNorm: для каждого вектора (строки) приводит значения
  к среднему = 0 и дисперсии = 1:

  LayerNorm(x) = γ · (x − μ) / √(σ² + ε) + β

  μ = среднее по 4096 элементам вектора
  σ² = дисперсия
  γ, β = обучаемые параметры (по 4096 каждый)
  ε = 1e-5 (для численной стабильности)

┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
┃  ЭТАП 5D. FEED-FORWARD NETWORK — «ТИХОЕ РАЗМЫШЛЕНИЕ»                        ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

  ╔═════════════════════════════════════════════════════════════════════════╗
  ║  АНАЛОГИЯ: Внутренний монолог                                         ║
  ║                                                                       ║
  ║  Attention — это совещание (токены общаются).                         ║
  ║  FFN — это когда после совещания ты садишься за стол                  ║
  ║  и ОДИН обдумываешь услышанное, применяя свои знания.                ║
  ╚═════════════════════════════════════════════════════════════════════════╝

  Двухслойная нейросеть, применяемая к каждому токену НЕЗАВИСИМО:

  FFN(x) = GeLU(x · W₁ + b₁) · W₂ + b₂

  W₁ ∈ ℝ^(4096 × 16384)  — расширение в 4×
  W₂ ∈ ℝ^(16384 × 4096)  — сжатие обратно

  GeLU — функция активации (плавный аналог ReLU):
  GeLU(x) = x · Φ(x)  где Φ — функция нормального распределения

  Визуально:

  ┌──────────────────────────────────────────────────────────────────┐
  │                                                                  │
  │  Вектор токена (4096)                                            │
  │       │                                                          │
  │       ▼                                                          │
  │  ┌─────────────────────────────────────────────┐                │
  │  │  Линейный слой 1: × W₁ + b₁                │                │
  │  │  4096 → 16384  (расширение)                  │                │
  │  └─────────────┬───────────────────────────────┘                │
  │                │                                                 │
  │                ▼                                                 │
  │  ┌─────────────────────────────────────────────┐                │
  │  │  GeLU (активация — вносит нелинейность)     │                │
  │  └─────────────┬───────────────────────────────┘                │
  │                │                                                 │
  │                ▼                                                 │
  │  ┌─────────────────────────────────────────────┐                │
  │  │  Линейный слой 2: × W₂ + b₂                │                │
  │  │  16384 → 4096  (сжатие)                      │                │
  │  └─────────────┬───────────────────────────────┘                │
  │                │                                                 │
  │                ▼                                                 │
  │  Вектор токена (4096) — обновлённый                             │
  │                                                                  │
  └──────────────────────────────────────────────────────────────────┘

  Зачем расширение? В «узком» пространстве (4096) информация
  спрессована. Расширяя до 16384, модель «разворачивает» знания
  в большем пространстве, обрабатывает, и сжимает обратно.

  Снова skip connection:
  X_out = LayerNorm(X_attn + FFN(X_attn))

  ШАГ 7.1: Temperature
  ─────────────────────

  Формула: logit_scaled = logit / T

  ┌──────────────────────────────────────────────────────────────────┐
  │                                                                  │
  │  T = 0.1 (холодная — предсказуемо):                             │
  │                                                                  │
  │  "Париж":  8.74 / 0.1 = 87.4   ← ДОМИНИРУЕТ                   │
  │  "Лондон": 4.12 / 0.1 = 41.2                                    │
  │  → почти 100% "Париж"                                           │
  │                                                                  │
  │  T = 1.0 (нейтральная — без изменений):                         │
  │                                                                  │
  │  "Париж":  8.74 / 1.0 = 8.74                                    │
  │  "Лондон": 4.12 / 1.0 = 4.12                                    │
  │  → "Париж" вероятнее, но есть шанс у других                     │
  │                                                                  │
  │  T = 2.0 (горячая — креативно):                                  │
  │                                                                  │
  │  "Париж":  8.74 / 2.0 = 4.37                                    │
  │  "Лондон": 4.12 / 2.0 = 2.06                                    │
  │  → разница маленькая, выбор почти случайный                      │
  │                                                                  │
  └──────────────────────────────────────────────────────────────────┘

  Визуально:

  Вероятность ▲
             │
  T=0.1:     │ █████████████████████████  "Париж"
             │ ░                          "Лондон"
             │                            (всё остальное ≈ 0)
             │
  T=1.0:     │ ████████████████           "Париж"
             │ █████                      "Лондон"
             │ ███                        "Берлин"
             │ █                          остальные
             │
  T=2.0:     │ ██████████                 "Париж"
             │ ███████                    "Лондон"
             │ ██████                     "Берлин"
             │ ████                       "город"
             │ ███                        другие
             └───────────────────────────────────────→


  ШАГ 7.2: Top-K
  ───────────────

  Оставляем только K токенов с наибольшими логитами.
  Пусть K = 5:

  ┌──────────────────────────────────────────────────────────────────┐
  │  ДО Top-K (100 000 кандидатов):                                  │
  │  "Париж"=8.74, "Лондон"=4.12, "Берлин"=3.89, "город"=2.45,    │
  │  "Рим"=2.31, "столица"=1.87, ..., "xyz"=−9.87                  │
  │                                                                  │
  │  ПОСЛЕ Top-K (K=5):                                              │
  │  "Париж"=8.74, "Лондон"=4.12, "Берлин"=3.89,                   │
  │  "город"=2.45, "Рим"=2.31                                       │
  │                                                                  │
  │  Остальные 99 995 — отброшены.                                   │
  └──────────────────────────────────────────────────────────────────┘


  ШАГ 7.3: Top-P (Nucleus Sampling)
  ──────────────────────────────────

  Из оставшихся берём минимальный набор с суммарной P ≥ 0.95:

  ┌──────────────────────────────────────────────────────────────────┐
  │                                                                  │
  │  После softmax по 5 кандидатам:                                  │
  │                                                                  │
  │  "Париж"   → 72%   ← накопленная: 72%  ✓ берём                 │
  │  "Лондон"  → 14%   ← накопленная: 86%  ✓ берём                 │
  │  "Берлин"  → 10%   ← накопленная: 96%  ✓ берём (перешли 95%)   │
  │  "город"   →  3%   ← отбрасываем                                │
  │  "Рим"     →  1%   ← отбрасываем                                │
  │                                                                  │
  │  Итого: 3 кандидата ("Париж", "Лондон", "Берлин")               │
  │                                                                  │
  └──────────────────────────────────────────────────────────────────┘


  ШАГ 7.4: Пересчёт softmax по финалистам
  ─────────────────────────────────────────

  softmax(xᵢ) = eˣⁱ / Σⱼ eˣʲ

  ┌──────────────────────────────────────────────────────────────────┐
  │                                                                  │
  │  "Париж"   →  75%   ██████████████████████████████████████       │
  │  "Лондон"  →  15%   ███████                                     │
  │  "Берлин"  →  10%   █████                                       │
  │                      ───────────────────────────────────────     │
  │                      Сумма: 100%                                 │
  │                                                                  │
  └──────────────────────────────────────────────────────────────────┘

  ┌────────────────────────────────┬───────────────────────────┐

  │                                │                           │

  │                                │                           │

┃  ВСЕ ФОРМУЛЫ В ОДНОМ МЕСТЕ                                                   ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

  Эмбеддинг:        x = E[token_id]
  Позиция:          PE(pos,2i) = sin(pos/10000^(2i/d))
  Вход:             X = Embedding + PositionalEncoding

  Attention:        Attention(Q,K,V) = softmax(Q·Kᵀ/√d_k + Mask)·V
  Q, K, V:          Q = X·W_Q,  K = X·W_K,  V = X·W_V
  Multi-Head:       MH = Concat(head₁,...,headₙ)·W_O

  FFN:              FFN(x) = GeLU(x·W₁ + b₁)·W₂ + b₂
  Layer:            X' = LN(X + MH(X)),  X'' = LN(X' + FFN(X'))
  LayerNorm:        LN(x) = γ·(x−μ)/√(σ²+ε) + β

  Логиты:           logits = h · Eᵀ
  Temperature:      logits_scaled = logits / T
  Softmax:          P(i) = eˡᵒᵍⁱᵗⁱ / Σⱼ eˡᵒᵍⁱᵗʲ
  Loss (обучение):  L = −log(P(correct_token))
  Обновление:       w = w − lr × ∂L/∂w
