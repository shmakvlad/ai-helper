# Документирование кодовой базы

## Что это делает
Создавайте исчерпывающую документацию для любой кодовой базы. Генерируйте README, руководства по архитектуре, документацию API и встроенные комментарии к коду — понятные для начинающих и соответствующие лучшим практикам.

## Быстрый старт

### Шаг 1: Перейдите в ваш проект
```bash
cd ~/your-project
```

### Шаг 2: Скачайте шаблон
Нажмите «Скачать» выше, затем выполните:
```bash
mv ~/Downloads/CLAUDE.md ./
```

### Шаг 3: Создайте документацию
```bash
claude
```
Затем попросите: «Создай исчерпывающую документацию для этого проекта»

---

## Типы документации

### README-документация
Для корневых каталогов и основных модулей:
- Что это делает
- Быстрый старт
- Структура проекта
- Ключевые концепции
- Типовые задачи
- Устранение неполадок

### Документация архитектуры
Для сложных проектов:
- Дизайн системы
- Структура каталогов
- Поток данных
- Принятые решения
- Зависимости модулей
- Точки расширения

### Документация API
Для HTTP-эндпоинтов и публичных интерфейсов:
- Описание эндпоинтов
- Аутентификация
- Форматы запросов и ответов
- Примеры использования
- Обработка ошибок

### Комментарии к коду
Для сложной логики:
- Документация функций
- Пояснения бизнес-логики
- Обработка граничных случаев

## Основные принципы

| Принцип | Описание |
|---------|----------|
| Начинайте с «Зачем» | Объясняйте назначение до деталей реализации |
| Постепенное раскрытие | Подавайте информацию слоями: просто → сложно |
| Контекст | Объясняйте зачем существует код, а не только что он делает |
| Примеры | Конкретное использование для каждой концепции |
| Без предположений | Определяйте термины, избегайте жаргона |
| Визуальные материалы | Диаграммы, блок-схемы, деревья файлов |
| Быстрый результат | Помогайте пользователям достичь успеха за 5 минут |

## Советы

- **Сначала анализируйте**: Попросите Claude изучить кодовую базу перед документированием
- **Приоритет точкам входа**: Начинайте с главных файлов и двигайтесь вглубь
- **Единообразное форматирование**: Помогает читателям ориентироваться в документации
- **Рабочие примеры**: Проверяйте фрагменты кода перед документированием
