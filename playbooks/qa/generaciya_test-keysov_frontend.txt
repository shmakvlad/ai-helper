ГЕНЕРАЦИЯ ТЕС-КЕЙСОВ ОСНОВЫВАЯСЬ НА КОДОВОЙ БАЗЕ FRONTEND РЕПОЗИТОРИЯ

Цель: Составить исчерпывающий список тест-кейсов на основе кодовой базы
frontend-приложения, без наличия требований от аналитиков.

Этап 1 — Анализ репозитория:

Ты — старший QA-инженер. Перейди по ссылке на репозиторий: [ССЫЛКА НА РЕПОЗИТОРИЙ]

Основываясь на кодовой базе, проанализируй следующее:
    1. Изучи структуру проекта: определи компоненты, страницы, роутинг
       и их взаимосвязи (React/Vue/Angular и т.д.).
    2. Найди все пользовательские сценарии: формы, кнопки, навигация,
       модальные окна, переходы между страницами.
    3. Проанализируй работу с состоянием: глобальный стейт (Redux, Vuex, Zustand),
       локальный стейт компонентов, передача данных через props.
    4. Изучи валидацию на стороне клиента: проверки полей форм,
       сообщения об ошибках, условия блокировки кнопок.
    5. Определи взаимодействие с API: какие эндпоинты вызываются,
       как обрабатываются ответы, ошибки и состояния загрузки.
    6. Выяви условную логику отображения: права доступа, роли пользователей,
       feature flags, адаптивность под разные устройства.

Этап 2 — Генерация тест-кейсов:

На основе проведённого анализа сгенерируй исчерпывающий список тест-кейсов.

Включи следующие категории:
    1. Позитивные сценарии — корректный ввод данных, успешная отправка форм,
       ожидаемая навигация и отображение контента.
    2. Негативные сценарии — некорректный ввод, пустые обязательные поля,
       неверные форматы данных, сообщения об ошибках.
    3. Граничные значения — минимум/максимум символов в полях,
       пустые списки, одиночные элементы, очень длинные строки.
    4. UI/UX проверки — корректность текстов, состояния кнопок (disabled/enabled),
       отображение лоадеров, пустых состояний и сообщений об ошибках.
    5. Авторизация и роли — доступ к страницам без авторизации,
       отображение элементов в зависимости от роли пользователя.
    6. Работа с API — отображение данных из ответа, поведение при ошибке сети,
       поведение при таймауте, корректность отображения состояния загрузки.
    7. Крайние случаи — спецсимволы в полях, XSS-попытки, быстрые повторные
       клики, одновременные действия пользователя.

Оформи результат в виде таблицы со столбцами:
    | ID | Категория | Описание теста | Шаги | Ожидаемый результат |
