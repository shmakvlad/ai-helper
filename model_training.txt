  Обучение нейросети — это процесс подбора миллиардов чисел (параметров)
  таким образом, чтобы модель хорошо предсказывала следующее слово
  в тексте.

  Модель ничего не «запоминает» буквально. Она настраивает свои
  внутренние числа так, что при получении входного текста на выходе
  появляется статистически правдоподобное продолжение.

  ╔═══════════════════════════════════════════════════════════════════════╗
  ║  АНАЛОГИЯ                                                           ║
  ║                                                                     ║
  ║  Представь, что ты учишь ребёнка играть на пианино.                 ║
  ║  Ты не заставляешь его запомнить координаты каждого пальца          ║
  ║  для каждой ноты. Вместо этого он играет тысячи раз,               ║
  ║  каждый раз чуть-чуть корректируя движения. В итоге                ║
  ║  пальцы «сами знают», куда нажимать — это не память,               ║
  ║  а натренированные рефлексы.                                        ║
  ║                                                                     ║
  ║  Параметры модели — это «мышечная память» пианиста.                 ║
  ║  Не ноты, а настроенные рефлексы.                                   ║
  ╚═══════════════════════════════════════════════════════════════════════╝


┌──────────────────────────────────────────────────────────────────────────────┐
│  1.2  ТРИ СТАДИИ ОБУЧЕНИЯ                                                   │
└──────────────────────────────────────────────────────────────────────────────┘


  ── СТАДИЯ 1: Pre-training (предобучение) ──────────────────────────────

  Это главная и самая дорогая стадия. Занимает недели/месяцы
  на тысячах GPU, стоит десятки и сотни миллионов долларов.

  ЧТО ПРОИСХОДИТ:

  1) Берётся гигантский корпус текста (терабайты):
     книги, статьи, код, Википедия, форумы, веб-страницы.

  2) Текст разбивается на последовательности токенов.

  3) Модели показывают начало фразы и просят предсказать
     следующий токен.

  Пример одной обучающей итерации:

  ┌────────────────────────────────────────────────────────────────────┐
  │                                                                    │
  │  Вход:   "Столица Франции —"                                      │
  │                                                                    │
  │  Модель предсказывает распределение вероятностей:                  │
  │    "Париж"   → 0.03   (модель пока не уверена)                    │
  │    "город"   → 0.05                                                │
  │    "большой" → 0.04                                                │
  │    "the"     → 0.02                                                │
  │    ...100 000 других токенов...                                    │
  │                                                                    │
  │  Правильный ответ: "Париж" (вероятность должна быть ~1.0)         │
  │                                                                    │
  │  Ошибка (loss): модель дала "Париж" только 0.03 — это плохо.     │
  │  Loss = −log(0.03) = 3.51  (чем ниже, тем лучше)                 │
  │                                                                    │
  └────────────────────────────────────────────────────────────────────┘

  4) Вычисляется ошибка (loss) — насколько предсказание
     отличается от правильного ответа.

     Используется cross-entropy loss:
     Loss = −log(P(правильный токен))

     Если модель дала правильному токену вероятность 0.03:
     Loss = −log(0.03) = 3.51 (высокая ошибка)

     Если модель дала 0.95:
     Loss = −log(0.95) = 0.05 (низкая ошибка — модель уверена)

  5) Backpropagation (обратное распространение ошибки):

     Это математический алгоритм, который вычисляет, КАК ИМЕННО
     нужно изменить КАЖДЫЙ из миллиардов параметров, чтобы
     ошибка уменьшилась.

     Он работает «от конца к началу»:
     ┌─────────────────────────────────────────────────────────┐
     │  Выход (ошибка)                                         │
     │    ↓ насколько виноват последний слой?                   │
     │  Слой 96                                                │
     │    ↓ насколько виноват предпоследний?                    │
     │  Слой 95                                                │
     │    ↓ ...                                                │
     │  Слой 1                                                 │
     │    ↓ насколько виновата таблица эмбеддингов?             │
     │  Эмбеддинги                                             │
     │                                                         │
     │  Результат: для КАЖДОГО параметра вычислен ГРАДИЕНТ —  │
     │  направление и величина нужного изменения.              │
     └─────────────────────────────────────────────────────────┘

     Математически это цепное правило (chain rule) из матанализа:
     ∂Loss/∂w = ∂Loss/∂output · ∂output/∂hidden · ... · ∂hidden/∂w

     Для каждого параметра w вычисляется его вклад в ошибку.

  6) Обновление параметров (optimizer step):

     Каждый параметр сдвигается в направлении, ПРОТИВОПОЛОЖНОМ
     градиенту (чтобы уменьшить ошибку):

     w_новый = w_старый − learning_rate × градиент

     learning_rate (скорость обучения) — маленькое число (~0.0001),
     которое определяет размер шага. Слишком большой шаг —
     модель «перепрыгнет» оптимум. Слишком маленький — обучение
     займёт вечность.

     Используются продвинутые оптимизаторы (AdamW, LAMB),
     которые адаптируют скорость для каждого параметра отдельно.

  7) Повторить триллионы раз.

     На каждой итерации обрабатывается не один пример,
     а batch (пакет) из тысяч примеров параллельно.

     Типичные масштабы:
     • Кол-во токенов в обучающих данных: 1–15 триллионов
     • Кол-во параметров модели: 70–1000+ миллиардов
     • Время обучения: 1–6 месяцев
     • Оборудование: 1000–30 000 GPU (A100/H100)
     • Стоимость: $10M–$500M+


  Схема одной итерации pre-training:

  ┌────────────────────────────────────────────────────────────────────┐
  │                                                                    │
  │  Текст ──→ Токенизация ──→ Эмбеддинги ──→ Трансформер ──→ Логиты│
  │                                                           │       │
  │                                                           ▼       │
  │                                                    Предсказание:  │
  │                                                    "Париж" = 0.03 │
  │                                                           │       │
  │  Правильный ответ: "Париж" ──→ Loss = −log(0.03) = 3.51 │       │
  │                                           │               │       │
  │                                           ▼               │       │
  │                                    Backpropagation        │       │
  │                                    (градиенты для         │       │
  │                                     каждого параметра)    │       │
  │                                           │               │       │
  │                                           ▼               │       │
  │                                    Обновление параметров  │       │
  │                                    w = w − lr × gradient  │       │
  │                                           │               │       │
  │                                           ▼               │       │
  │                                    Следующий batch ───────┘       │
  │                                                                    │
  └────────────────────────────────────────────────────────────────────┘


  ВАЖНО: модель учится не на одном слове, а на КАЖДОЙ ПОЗИЦИИ
  последовательности одновременно:

  Вход:     "Столица  Франции  —    Париж  ,   красивый  город"
  Цели:     "Франции" "—"      "Париж" ","  "красивый" "город" "."

  Для каждой позиции вычисляется свой loss, и все они усредняются.
  Одна последовательность из 2048 токенов даёт ~2048 обучающих
  сигналов. Это делает обучение очень эффективным.


  ── СТАДИЯ 2: Fine-tuning + RLHF ──────────────────────────────────────

  После pre-training модель умеет продолжать текст, но не умеет
  «общаться». Если её спросить "Какая столица Франции?", она может
  ответить "Какая столица Германии? Какая столица Италии?" —
  просто продолжая паттерн вопросов.

  Шаг 2a: Supervised Fine-Tuning (SFT)
  ──────────────────────────────────────
  Люди-аннотаторы пишут тысячи пар «вопрос → идеальный ответ».
  Модель дообучается на этих парах тем же способом (предсказание
  следующего токена), но теперь на данных в формате диалога.

  Пример обучающей пары:
  ┌────────────────────────────────────────────────────────────┐
  │  User: Какая столица Франции?                              │
  │  Assistant: Столица Франции — Париж. Это крупнейший город │
  │  страны, расположенный на реке Сене.                       │
  └────────────────────────────────────────────────────────────┘

  Шаг 2b: RLHF (Reinforcement Learning from Human Feedback)
  ──────────────────────────────────────────────────────────────
  1) Модели дают один и тот же вопрос, она генерирует
     несколько вариантов ответа (например 4 штуки).

  2) Люди ранжируют ответы от лучшего к худшему.

  3) На этих рангах обучается отдельная Reward Model —
     модель, которая предсказывает человеческую оценку.

  4) Основная модель дообучается через PPO (Proximal Policy
     Optimization) — алгоритм обучения с подкреплением,
     который максимизирует «награду» от Reward Model,
     одновременно не уходя слишком далеко от SFT-версии.

  Результат: модель предпочитает полезные, безопасные,
  честные ответы — не потому что «понимает» это,
  а потому что параметры настроены на такой вывод.


  ── СТАДИЯ 3: Оценка и итерации ───────────────────────────────────────

  Модель тестируют на бенчмарках (MMLU, HumanEval, GSM8K и др.)
  и в реальных диалогах. Находят слабые места, корректируют:
  • данные (добавляют примеры для слабых областей)
  • гиперпараметры (learning rate, batch size, warmup)
  • архитектуру (если нужно)

  Цикл повторяется до приемлемого качества.

┌──────────────────────────────────────────────────────────────────────────────┐
│  2.1  ЧТО ТАКОЕ 4096                                                        │
└──────────────────────────────────────────────────────────────────────────────┘

  4096 — это РАЗМЕРНОСТЬ СКРЫТОГО ПРЕДСТАВЛЕНИЯ (hidden dimension,
  она же d_model). Это число определяет, сколько чисел используется
  для описания каждого токена внутри модели.

  Когда мы говорим «эмбеддинг — вектор из 4096 чисел», мы имеем
  в виду именно это число.

  Это НЕ фиксированная константа. Это ГИПЕРПАРАМЕТР — выбор
  архитекторов модели:

  ┌────────────────────────────────────────────────────────────────────┐
  │  Модель                │  d_model (скрытая размерность)           │
  ├────────────────────────┼─────────────────────────────────────────  │
  │  GPT-2 Small           │     768                                  │
  │  GPT-2 Large           │   1 280                                  │
  │  GPT-3 (175B)          │  12 288                                  │
  │  LLaMA-2 7B            │   4 096                                  │
  │  LLaMA-2 70B           │   8 192                                  │
  │  Mistral 7B            │   4 096                                  │
  │  Claude / GPT-4 класс  │   8 192 – 16 384+ (предположительно)    │
  └────────────────────────────────────────────────────────────────────┘

  4096 — просто популярное значение для моделей размера ~7B
  параметров. Для более крупных моделей оно больше.

  Почему именно степени двойки (4096 = 2¹², 8192 = 2¹³)?
  Потому что GPU-ядра оптимизированы для работы с размерами,
  кратными 2, 8, 16, 64 — это даёт максимальную скорость
  матричных умножений.


┌──────────────────────────────────────────────────────────────────────────────┐
│  2.2  ГДЕ ВСТРЕЧАЕТСЯ 4096 ВНУТРИ МОДЕЛИ                                    │
└──────────────────────────────────────────────────────────────────────────────┘

  Число 4096 (d_model) пронизывает ВСЮ архитектуру.
  Оно определяет размер почти каждой матрицы в модели:


  А) ТАБЛИЦА ЭМБЕДДИНГОВ
  ───────────────────────
  Размер: vocab_size × d_model = 100 000 × 4 096

  Каждый токен в словаре имеет свой вектор из 4096 чисел.
  Это «адрес» токена в пространстве смыслов.

  Кол-во параметров в таблице: 100 000 × 4 096 = 409 600 000
  (почти полмиллиарда — только в одной таблице!)


  Б) ПОЗИЦИОННЫЕ ЭМБЕДДИНГИ
  ──────────────────────────
  Размер: max_seq_length × d_model = 4 096 × 4 096
  (если максимальная длина контекста тоже 4096 токенов —
   это совпадение, не связанные числа)

  Каждая позиция в последовательности имеет свой вектор из 4096 чисел.


  В) МАТРИЦЫ Q, K, V В КАЖДОМ СЛОЕ ATTENTION
  ────────────────────────────────────────────
  Каждый слой содержит три матрицы проекции:

  W_q: d_model × d_model = 4 096 × 4 096 = 16 777 216 параметров
  W_k: d_model × d_model = 4 096 × 4 096 = 16 777 216 параметров
  W_v: d_model × d_model = 4 096 × 4 096 = 16 777 216 параметров
  W_o: d_model × d_model = 4 096 × 4 096 = 16 777 216 параметров

  Итого на attention одного слоя: ~67 млн параметров

  При Multi-Head Attention (например 32 головы):
  Каждая голова работает с размерностью d_model / n_heads = 4096 / 32 = 128
  Но суммарно все головы дают те же 4096.


  Г) FEED-FORWARD NETWORK В КАЖДОМ СЛОЕ
  ──────────────────────────────────────
  FFN расширяет и сжимает вектор:

  W₁: d_model × d_ff    = 4 096 × 16 384 = 67 108 864 параметров
  W₂: d_ff    × d_model = 16 384 × 4 096 = 67 108 864 параметров

  d_ff обычно = 4 × d_model = 4 × 4096 = 16 384
  (это тоже гиперпараметр, но 4× — стандартное соотношение)

  Итого на FFN одного слоя: ~134 млн параметров


  Д) ФИНАЛЬНАЯ ПРОЕКЦИЯ (логиты)
  ──────────────────────────────
  Размер: d_model × vocab_size = 4 096 × 100 000

  Часто используется та же матрица, что и таблица эмбеддингов
  (weight tying), только транспонированная. Это экономит память.


┌──────────────────────────────────────────────────────────────────────────────┐
│  2.3  КАРТА ПАРАМЕТРОВ ОДНОГО СЛОЯ                                           │
└──────────────────────────────────────────────────────────────────────────────┘

  Вот полная раскладка параметров для одного слоя трансформера
  при d_model = 4096:

  ┌──────────────────────────────────────────────────────────────┐
  │                                                              │
  │  ОДИН СЛОЙ ТРАНСФОРМЕРА                                      │
  │                                                              │
  │  Multi-Head Attention:                                       │
  │  ├── W_q:  4096 × 4096  =  16 777 216  параметров           │
  │  ├── W_k:  4096 × 4096  =  16 777 216  параметров           │
  │  ├── W_v:  4096 × 4096  =  16 777 216  параметров           │
  │  ├── W_o:  4096 × 4096  =  16 777 216  параметров           │
  │  └── Итого attention:      67 108 864  (~67M)                │
  │                                                              │
  │  Feed-Forward:                                               │
  │  ├── W₁:  4096 × 16384  =  67 108 864  параметров           │
  │  ├── W₂:  16384 × 4096  =  67 108 864  параметров           │
  │  └── Итого FFN:           134 217 728  (~134M)               │
  │                                                              │
  │  Layer Norm (×2):             2 × 4096 × 2  =  16 384       │
  │                                                              │
  │  ═══════════════════════════════════════════                  │
  │  ИТОГО НА ОДИН СЛОЙ:     ~201 000 000 (~201M)               │
  │                                                              │
  └──────────────────────────────────────────────────────────────┘

  Для модели с 32 слоями:
  32 × 201M = ~6.4 миллиарда + эмбеддинги (~0.4B) ≈ 7B параметров

  Именно так получается модель класса «7B» (LLaMA-2 7B, Mistral 7B).

  Увеличивая d_model и/или количество слоёв, получают модели
  13B, 34B, 70B, 175B, 540B и далее.


┌──────────────────────────────────────────────────────────────────────────────┐
│  2.4  ИНТУИЦИЯ: ЗАЧЕМ СТОЛЬКО ИЗМЕРЕНИЙ?                                    │
└──────────────────────────────────────────────────────────────────────────────┘

  Почему 4096, а не 10 или 100?

  Каждое число в векторе — это одно «измерение смысла».
  Чем больше измерений, тем больше нюансов может кодировать модель.

  ╔═══════════════════════════════════════════════════════════════════════╗
  ║  АНАЛОГИЯ «ОПИСАНИЕ ЧЕЛОВЕКА»                                       ║
  ║                                                                     ║
  ║  Если описывать человека одним числом (рост) — все люди              ║
  ║  одного роста неразличимы.                                           ║
  ║                                                                     ║
  ║  Два числа (рост + вес) — уже лучше, но всё ещё грубо.             ║
  ║                                                                     ║
  ║  10 чисел (рост, вес, возраст, цвет глаз, ...) — можно             ║
  ║  различить большинство людей.                                        ║
  ║                                                                     ║
  ║  4096 чисел — можно закодировать невероятное количество              ║
  ║  нюансов: не только «что это за слово», но и «в каком               ║
  ║  контексте оно обычно встречается», «какие эмоции несёт»,           ║
  ║  «насколько формально», «к какой предметной области относится»,     ║
  ║  и тысячи других неименованных характеристик.                        ║
  ╚═══════════════════════════════════════════════════════════════════════╝

  Важно: ни одно из 4096 измерений не имеет подписи. Мы не знаем,
  какое измерение отвечает за «одушевлённость», а какое за «формальность».
  Эти направления возникают в процессе обучения и переплетены друг
  с другом. Интерпретация отдельных нейронов — активная область
  исследований (mechanistic interpretability).


┌──────────────────────────────────────────────────────────────────────────────┐
│  2.5  КАК d_model ВЛИЯЕТ НА КАЧЕСТВО                                        │
└──────────────────────────────────────────────────────────────────────────────┘

  Больше d_model = больше «ёмкости» модели для хранения знаний
  и нюансов. Но есть закон убывающей отдачи:

  ┌────────────────────────────────────────────────────────────────────┐
  │  d_model     │  Параметры (при 32 слоях) │  Что может             │
  ├──────────────┼──────────────────────────┼────────────────────────  │
  │     768      │       ~0.1B              │  Простые задачи,         │
  │              │                          │  базовая грамматика      │
  │              │                          │                          │
  │   2 048      │       ~1.4B              │  Связный текст,          │
  │              │                          │  базовые знания          │
  │              │                          │                          │
  │   4 096      │       ~7B                │  Хорошие знания,         │
  │              │                          │  рассуждения,            │
  │              │                          │  код на уровне           │
  │              │                          │  джуниора                │
  │              │                          │                          │

  Теперь объединим обе темы — как обучение настраивает
  все эти матрицы с числом 4096:

  ДО ОБУЧЕНИЯ:
  ────────────
  Все матрицы (W_q, W_k, W_v, W₁, W₂, таблица эмбеддингов...)
  инициализированы случайными числами, близкими к нулю.

  Таблица эмбеддингов 100 000 × 4096 — случайный шум.
  Матрицы Q, K, V — случайный шум.
  FFN — случайный шум.

  Модель выдаёт бессмысленные предсказания.

  ПРОЦЕСС ОБУЧЕНИЯ:
  ──────────────────
  Каждый batch данных → forward pass (прямой проход через все слои)
  → loss → backpropagation → обновление ВСЕХ матриц.

  За триллионы итераций:

  • Таблица эмбеддингов (100K × 4096): числа перестраиваются так,
    что «кот» и «кошка» получают близкие 4096-мерные векторы,
    а «кот» и «бетон» — далёкие.

  • Матрицы W_q, W_k (каждая 4096 × 4096): настраиваются так,
    что скалярное произведение Q("прилагательное") · K("существительное")
    даёт высокую оценку — модель учится, что прилагательные
    должны «обращать внимание» на существительные.

  • FFN (4096 → 16384 → 4096): веса настраиваются так, что
    определённые комбинации активаций кодируют факты:
    «Если на входе вектор, похожий на 'столица + Франции',
    усилить компоненты, связанные с 'Париж'».

  • Всё это происходит одновременно, совместно, через градиенты.

  ПОСЛЕ ОБУЧЕНИЯ:
  ───────────────
  Каждое из ~7 миллиардов чисел стоит на своём месте.
  Ни одно число по отдельности не означает ничего.
  Но все вместе они определяют, как модель превращает
  «Расскажи анекдот про кота» в «Идёт кот по крыше...».

  ИТОГ
══════════════════════════════════════════════════════════════════════════════════

  • Обучение — это триллионы циклов «предскажи → ошибка →
    backpropagation → подкрути параметры».

  • 4096 — это d_model: размерность вектора, описывающего каждый токен.
    Гиперпараметр, выбранный архитекторами. Определяет «ёмкость» модели.

  • Это число задаёт размер КАЖДОЙ матрицы весов в модели:
    эмбеддингов, Q/K/V, FFN, финальной проекции.

  • Обучение настраивает все эти матрицы совместно, через градиенты,
    так что 7 миллиардов случайных чисел превращаются в систему,
    которая генерирует осмысленный текст.

  • Больше d_model → больше нюансов → лучше качество,
    но квадратичный рост стоимости.
